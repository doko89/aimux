# aimux — AI API Gateway

Anthropic ↔ OpenAI protocol translation gateway with multi-provider routing, model aggregation, and interactive setup TUI.

Version 0.1.1 — [Releases](https://github.com/doko89/aimux/releases)

## Features

- **Protocol bridge** — Accepts Anthropic Messages API, converts to OpenAI Chat Completions
- **Interactive setup** — Bubbletea TUI to manage all configuration (`aimux setup`)
- **Multi-provider** — Routes to any OpenAI-compatible backend (OpenAI, DeepSeek, Mimo, OpenRouter, etc.)
- **ChatGPT/Codex provider** — Use your ChatGPT Plus/Pro subscription via OAuth device flow
- **Model aggregation** — Virtual models that distribute traffic via weighted, fallback, or round-robin
- **Circuit breaker** — Automatic failover with health checks and cooldown
- **Reasoning** — Bidirectional Anthropic thinking ↔ OpenAI reasoning_content conversion
- **Streaming** — Full SSE streaming with real-time format conversion
- **Client API keys** — Manage named client credentials with auto-generate

## CLI

```bash
aimux setup               # Interactive configuration TUI
aimux server              # Start the API gateway server
aimux login chatgpt       # Login with ChatGPT account (OAuth device flow)
aimux login chatgpt -f    # Force re-login
aimux version             # Show version
aimux help                # Show help
```

## Setup Wizard

```bash
aimux setup
```

Sidebar-based TUI with 4 pages:

| Tab | Description |
|-----|-------------|
| **Auth** | Manage providers: add/edit/remove, name/base URL/API Key, fetch models from API, test connection |
| **Aggregator** | Virtual models: name, strategy (weighted/fallback/round_robin), add provider/model pairs with search modal |
| **Credential** | Client API keys: add, name, generate random key (`ak-xxx`), expand/collapse inline editing |
| **Settings** | Gateway host/port, routing strategy, circuit breaker, rate limiting |

Generates a single `config.yaml` file with all configuration.

### ChatGPT/Codex Login

```bash
aimux login chatgpt
```

Opens a device OAuth flow — URL and code to enter in browser.
Token saved to `~/.aimux/chatgpt-auth.json` with auto-refresh.

## Configuration

Single file: `config.yaml`

```yaml
# aimux config — Generated by aimux setup

gateway:
  host: 0.0.0.0
  port: 8080
  debug: false

routing:
  strategy: weighted
  fallback_on_error: true
  max_retries: 2

circuit_breaker:
  failure_threshold: 5
  cooldown_seconds: 60
  health_check_interval: 30

rate_limiting:
  enabled: true
  rpm: 100
  burst: 20

client_keys:
  - name: default
    key: ak-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

providers:
  - name: mimo
    enabled: true
    base_url: https://token-plan-sgp.xiaomimimo.com/v1
    api_key: tp-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    model: mimo-v2.5
    available_models:
      - mimo-v2.5
      - mimo-v2.5-pro
    weight: 10
    priority: 6
    timeout: 120

model_aggregations:
  - name: codex
    strategy: weighted
    models:
      - provider: codex
        model: gpt-5.5
        weight: 100
```

## Quick Start

```bash
# Download from releases or build
go build -o aimux .

# One-time setup
./aimux setup

# Login with ChatGPT (optional, for Codex provider)
./aimux login chatgpt

# Start server
./aimux server
```

### From source

```bash
git clone https://github.com/doko89/aimux.git
cd aimux
go build -o aimux .
./aimux setup
```

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/v1/messages` | Anthropic Messages API (converted to OpenAI) |
| POST | `/v1/messages/count_tokens` | Token counting |
| POST | `/v1/chat/completions` | OpenAI Chat Completions API |
| POST | `/v1/responses` | OpenAI Responses API v2 |
| GET | `/v1/models` | List available models (aggregations only) |
| GET | `/health` | Health check |
| GET | `/health/providers` | Per-provider status |
| GET | `/admin/stats` | Admin statistics |

## Model Aggregations

Strategies: `weighted`, `fallback`, `round_robin`

```yaml
model_aggregations:
  - name: flash
    strategy: round_robin
    models:
      - provider: zai
        model: glm-4.7
        weight: 50
      - provider: mimo
        model: mimo-v2.5
        weight: 50

  - name: pro
    strategy: fallback
    models:
      - provider: mimo
        model: mimo-v2.5-pro
      - provider: zai
        model: glm-5.2
```

## Architecture

```
Client (Anthropic API)
    ↓ POST /v1/messages
aimux gateway
    ├─ Anthropic → OpenAI format conversion
    ├─ Model aggregation routing
    ├─ Circuit breaker failover
    └─ OpenAI → Anthropic format conversion
    ↓
Providers (OpenAI, Codex, DeepSeek, Mimo, etc.)
```

## Deploy (systemd)

```bash
GOOS=linux GOARCH=arm64 go build -o aimux-linux-arm64 .
scp aimux-linux-arm64 user@server:/apps/ai-router/aimux
ssh user@server "sudo systemctl restart ai-router"
```

Example systemd service:

```ini
[Unit]
Description=AI Router Gateway
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/apps/ai-router
ExecStart=/apps/ai-router/aimux server
Restart=always
RestartSec=5
LimitNOFILE=1048576

[Install]
WantedBy=multi-user.target
```
